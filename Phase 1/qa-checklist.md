# QA Checklist

Use this before any Phase 1 launch or design review.

## Content

- [ ] Company name is consistent.
- [ ] Address is verified.
- [ ] Email is verified.
- [ ] Phone number is added or intentionally omitted.
- [ ] Social links are verified.
- [ ] All six workforce metrics are present.
- [ ] Metrics use owner-approved numbers.
- [ ] Testimonials have permission and correct attribution.
- [ ] No fake client logos, awards, certifications, or guarantees are invented.
- [ ] CTAs are clear and action-oriented.

## Design

- [ ] First screen communicates what Americall does.
- [ ] Primary CTA is visible above the fold.
- [ ] Typography is modern, readable, and not overly italic/serif.
- [ ] Layout is not dominated by one dark color.
- [ ] Logo red is used as a restrained accent, not as a dominant background color.
- [ ] Cards/buttons have restrained radii.
- [ ] Spacing is generous and consistent.
- [ ] Images feel professional and human.
- [ ] No generic/cheesy stock imagery remains without approval.

## Responsive

- [ ] Works at 360px mobile width.
- [ ] Works at 390px mobile width.
- [ ] Works at 768px tablet width.
- [ ] Works at 1024px laptop width.
- [ ] Works at 1440px desktop width.
- [ ] Header/nav works on mobile.
- [ ] Text does not overflow containers.
- [ ] Form fields are easy to tap.
- [ ] Agent cards and audio controls work on mobile.

## Accessibility

- [ ] Semantic heading order.
- [ ] All interactive elements are keyboard reachable.
- [ ] Focus states are visible.
- [ ] Form fields have labels.
- [ ] Form errors are clear.
- [ ] Meaningful images have alt text.
- [ ] Decorative images have empty alt text.
- [ ] Color contrast is sufficient.
- [ ] Reduced motion is respected.
- [ ] Audio controls are not hover-only.
- [ ] AI-generated effects/state/form/audio code has been reviewed line by line.

## Performance

- [ ] Images are compressed.
- [ ] Use WebP/AVIF where appropriate.
- [ ] Audio files are compressed and below Cloudflare Pages asset size limits.
- [ ] Below-the-fold images are lazy-loaded.
- [ ] Unused JavaScript is avoided.
- [ ] Animations do not cause jank on mobile.
- [ ] Build output is checked.
- [ ] Core Web Vitals risks are reviewed.

## SEO

- [ ] Each page has a unique title.
- [ ] Each page has a meta description.
- [ ] Canonical URLs are set.
- [ ] Open Graph metadata exists.
- [ ] Sitemap exists.
- [ ] Robots.txt exists.
- [ ] Footer includes verified business details.
- [ ] Organization schema considered.
- [ ] Google Search Console plan exists.

## Forms

- [ ] Provider chosen.
- [ ] Submission recipient verified.
- [ ] Spam protection enabled.
- [ ] Success state works.
- [ ] Error state works.
- [ ] Required fields are sensible.
- [ ] No sensitive data is collected unnecessarily.
- [ ] Privacy note included if needed.

## Deployment

- [ ] GitHub repository connected.
- [ ] Cloudflare Pages build command confirmed.
- [ ] Cloudflare Pages output directory confirmed.
- [ ] Preview deployment tested.
- [ ] Production deployment tested.
- [ ] Custom domain connected.
- [ ] SSL active.
- [ ] Redirects configured if needed.

## Final Review

- [ ] Owner reviewed desktop.
- [ ] Owner reviewed mobile.
- [ ] Owner approved copy.
- [ ] Owner approved logo usage.
- [ ] Owner approved launch domain.
- [ ] Maintenance/update process explained.
